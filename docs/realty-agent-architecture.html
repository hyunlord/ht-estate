부동산 Agent — 시스템 아키텍처v1

조건으로 검색이 안 되는 정보(헬스장·전용면적·세대당 주차·신축 정도·강아지·구조)를 — 벌크로 구축한 결정론 레이어와 필요할 때만 긁어 캐싱하는 확률 레이어로 합치고, 모든 사실에 출처를 단다.

eager warm · build-time lazy read-through · query-time provenance · 출처 핵심 난이도
한 개의 저장소 · 두 가지 적재 모드 search = cache miss 로 트리거되는 lazy 적재. build와 search는 같은 store의 cold / warm path.
